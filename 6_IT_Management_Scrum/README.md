# 🚀 Scrum Project Experience — Social Media Web Application

This section highlights my experience working in an Agile/Scrum development environment as part of a team project. The goal of the project was to build a social media–style web application that allowed users to create, edit, delete, and view posts using a combination of frontend (HTML, CSS, JavaScript) and backend (PHP, MySQL) technologies.

Although the project is still in progress, the teamwork, planning process, iterative development, and Agile collaboration provided valuable experience closely aligned with real-world IT project workflows.

---

# 👨‍💻 My Role: Developer

I worked as a **Developer** on this project. My responsibilities included:

- Implementing JavaScript functionality for user interactions  
- Helping build UI components, forms, and dynamic page updates  
- Integrating PHP backend logic with frontend fetch() requests  
- Debugging issues between JavaScript and PHP endpoints  
- Supporting feature development across multiple sprints  
- Reviewing merge conflicts and ensuring updated code consistency  

This provided hands-on experience with both frontend and backend development in a collaborative environment.

---

# 🧩 Scrum Structure and Team Workflow

### 🟦 **Scrum Roles**
- **Product Owner** – Defined user stories, requirements, and priorities  
- **Scrum Master** – Facilitated meetings, assisted with blockers, and managed workflow  
- **Developers** (including myself) – Built features, integrated components, and resolved bugs  

### 🟨 **Product Backlog Items**
The backlog contained features such as:

- User post creation  
- Editing and deleting posts  
- Displaying posts dynamically  
- User-specific content and filtering  
- Backend database storage (PHP + MySQL)  
- API endpoints for CRUD operations  
- UI improvements and layout changes  

### 🟩 **Team Sprints**
The project progressed through several iterative sprint cycles:

#### **UI Foundation**
- Built the initial HTML structure  
- Created forms for creating posts  
- Added basic CSS styling  
- Implemented simple DOM manipulation in JavaScript  

#### **Local Data Handling**
- Implemented localStorage to mock saving posts  
- Allowed quick demonstration of features without a backend  
- Focused on rapid iteration and MVP functionality  

#### **Backend Integration**
- Added PHP scripts for create / edit / delete post operations  
- Set up MySQL tables for storing post data  
- Transitioned from localStorage to persistent database logic  
- Integrated frontend fetch() calls with PHP backend endpoints  

#### **Debugging & Feature Expansion**
- Resolved file version conflicts  
- Cleaned up inconsistent code  
- Improved error-handling and data validation  
- Tested frontend–backend communication  
- Stabilized the overall application flow

# 🔄 Applying Scrum in Practice

This project followed true Scrum principles:

### ✔ Iterative development  
Features were not built all at once; they evolved over several sprints.

### ✔ Incremental improvements  
We started with a minimal product (localStorage version) and improved it into a full backend system.

### ✔ Collaboration and communication  
Daily coordination helped ensure tasks were understood and blockers were removed.

### ✔ Transparency and adaptability  
We often had to adjust priorities or resolve unexpected issues, which mirrors real-world Agile workflows.

# ⚠️ Challenges & How We Overcame Them

### 🟣 Challenges
- Managing multiple versions of files across team members  
- Ensuring everyone worked on the *latest* code  
- Debugging communication between JavaScript and PHP  
- Time constraints and shifting priorities  
- Maintaining consistency between frontend and backend behaviors  

### 🟢 Lessons Learned
- The importance of version control (Git) and consistent naming conventions  
- Why communication matters within cross-functional teams  
- How frontend and backend systems must align in both logic and structure  
- The value of incremental development to reduce complexity  
- The need for clean code and frequent testing  

---

# 🔐 Security Considerations During Development

Even at a student project level, we applied several basic secure coding practices:

- Input validation (preventing empty or invalid posts)  
- Sanitizing data before inserting into the database  
- Using try/catch blocks in PHP for safer error handling  
- Avoiding trust in client-side data alone  
- Understanding the importance of authentication (future improvement)  

These principles form the foundation of secure web application development.
